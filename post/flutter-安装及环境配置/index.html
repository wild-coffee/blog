Flutter 安装及环境配置

June 20, 2022

Flutter 是 Google 开源的应用开发框架，仅通过一套代码库，就能构建精美的、原生平台编译的多平台应用。官方网站：https://flutter.cn/

安装及环境变量配置

最新的安装方法以开发文档为准：https://flutter.cn/docs/get-started/install/windows

下载最新的 SDK ：https://flutter.cn/docs/get-started/install，下载后，把解压得到的 flutter 目录复制到用户目录（C:\Users\Alan\flutter）

配置环境 Path 指定 SDK 路径

在用户环境变量 Path 中添加：%USERPROFILE%\flutter\bin，添加完成后命令行执行：where flutter dart，可工作即可

在用户环境变量中添加以下以下 2 个变量，以解决国内 flutter 升级、包更新等太慢的问题

PUB_HOSTED_URL=https://pub.flutter-io.cn
FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn

Flutter doctor 检查，安装 Android studio 及 Android SDK

在 Flutter 安装完成后，启动命令行执行：flutter doctor 检查，全部为绿色则表示 flutter 安装成功

  • 错误 1：Android toolchain - develop for Android devices.

    则需要安装 Android studio，下载地址：https://developer.android.google.cn/studio?hl=zh-cn

  • 错误 2：cmdline-tools component is missing

    则需要在 Android studio 中安装：SDK Manager->SDK Tools->Android SDK Command-line Tools(latest)

    安装完成后执行：flutter doctor --android-licenses

Flutter 升级

Flutter 支持 master，dev，beta，stable 等发布渠道，开发一般用 stable 渠道

flutter channel                     # 可以查看 Flutter 支持的发布渠道及当前使用的发布渠道
flutter channel <channel name>      # 切换 Flutter 的发布渠道
flutter upgrade                     # 升级当前渠道最新代码

如果只想升级管理项目依赖包，可以使用 flutter pub 命令（pub 命令需要进入到项目所在目录）

flutter pub upgrade                 # 按 pubspec.yaml 配置把所有依赖更新到最新的兼容版本
flutter pub get                     # 按 pubspec.yaml 配置下载项目所有依赖包
flutter clean                       # 清理项目所有依赖包及编译产生的相关文件
flutter pub outdated                # 查看需要升级的依赖包情况
dart fix --apply                    # 修复工程

解决 Running Gradle task ‘assembleDebug’… gradle sync failed 问题

打开 Windows 设置 > 网络和 Internet > 代理，找到本地代理的端口

打开 Android Studio 里的 Setting > Appearance & Behavior > System settings > HTTP Proxy

启用 Manual proxy configuration，启用 HTTP，配置 Host name 为代理 IP，同时设置代理端口

点击 Check connection，使用 dl.google.com 测试，成功则代表代理配置正确，点击应用确认，在菜单 File > Sync Project with Gradle Files
